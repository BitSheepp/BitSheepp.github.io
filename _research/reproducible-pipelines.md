This line explains how the work is kept stable. Citrus genomics and trait analysis quickly become hard to extend when scripts, data, and outputs are loosely connected, so part of the research effort goes into building analysis systems that remain clear and reusable.

## Research question

How should bioinformatics workflows be organized so citrus genomics and trait studies can be checked, extended, and handed off without losing clarity?

## Current focus

- stable project structure for genomics and transcriptomics
- repeatable analysis logic across related studies
- interpretable outputs for collaboration and reuse
- smoother handoff between projects, manuscripts, and collaborators

## Methods path

This line emphasizes:

1. clean data organization
2. documented analysis steps
3. reproducible outputs
4. workflows that can be reused across related citrus questions

## Representative role

- supports citrus disease-resistance and fruit-quality analyses
- reduces maintenance overhead across connected projects
- provides a transferable computational foundation beyond one single dataset
